### Blockchain and Smart Contracts
#### Ethereum * Introduced blockchain computation -> Ethereum Virtual Machine (EVM) * Introduced blockchain storage -> SWARM (under development) * Consensus: Ethash (modified PoW)
##### Blockchain Smart Contracts * Code that is **stored in**, **acts upon** and **executed on** the blockchain ##### Legally Binding Smart Contracts [2] * Code that **articulates**, **verifies**, and **enforces** an agreement between parties [2] Stark, J. ["Making Sense of Blockchain Smart Contracts"](https://www.coindesk.com/making-sense-smart-contracts) (2016).
#### Beware of the hype... [3] * Cryptocurrencies Market Cap: 135 Billion $ * Largest known attack ~ 500M $ ![Blockchain Hype]](./hype.png) [3] Bartoletti, M. and Pompianu, L., [An empirical analysis of smart contracts...](https://arxiv.org/pdf/1703.06322.pdf) (2017)
#### Smart Contracts in Ethereum (1) * Already existed in Bitcoin with Script (stack-based) * Ethereum extends with EVM (Turing-complete) * EVM bindings to high level smart conrtact languages ([Solidity](https://solidity.readthedocs.io/)) * Communication with RPC and WS * Transactions: RPCs that modify blockchain state
##### Smart Contracts in Ethereum (2) * Execution in EVM is fully replicated * Min processing time ~ Block time * Code execution has a [cost](https://docs.google.com/spreadsheets/d/1m89CVujrQe5LAFJ8-YAUCcNK950dUzMQPMJBxRtGCqs/edit#gid=0) ([gas](http://ethdocs.org/en/latest/contracts-and-transactions/account-types-gas-and-transactions.html)) * Communication with external world? (Events, Oracles) * Standardization -> ERCs
